1. 行为克隆 (Behavior Cloning)

专家在方块环境中，演示如何移动到钥匙、捡起钥匙、打开门、到达目标。将这些步骤中的 (s,a) 收集起来，当作我们的专家数据。

我们把专家的动作a当作标签，让 agent 在相同状态s 时，也输出正确的动作a。

与分类问题的计算一样

专家的动作a当作标签，为目标类别。我们让该类别可能性越来越大。

行为克隆只是单纯的动作模仿，不会有任何逻辑。

2. 逆强化学习(inverse reinforcement leanring)

MaxEnt IRL（最大熵逆强化学习）是一种从专家演示中推断奖励函数的算法。

牢记：从专家演示中推断奖励函数的算法。因为后面理论公式一堆，很容易不知道自己干啥去了。

2.1 理论

我们先详细说明奖励网络的理论。

首先，MaxEnt IRL 的目标是最大化专家轨迹的对数似然 (即找一个θ参数的模型，传入专家轨迹链τ，最大化返回概率值)：

D是我们的专家经验。 \[ Target(\theta) = \sum_{\tau \in D}\log P(\tau|\theta) \] 其中专家轨迹τ的概率计算如下 \[ P(\tau|\theta) = \exp(\theta^{\top}f(\tau))/Z(\theta) \] 对于公式中的θ和f，理解如下

理论上奖励是 \(R(τ)=θ^⊤f(τ)\)，需要手动设计特征 f，再学习权重 θ。但我们代码直接用神经网络替代了这整个过程。

好，回到刚才我们的概率计算中，我们最终目的是对最大化专家轨迹的对数似然进行求导 \[ P(\tau|\theta) = \exp(\theta^{\top}f(\tau))/Z(\theta) \] 其中Z为所有可能轨迹的概率求和，使我们的 概率P值归纳到0-1之间。 \[ Z(θ) = \sum_τ exp(θ^{\top} f(τ)) \] 现在我们有 N 条专家轨迹，目标是让这些轨迹在参数 θ 下出现的概率尽量大。 \[ Target(\theta) = \sum_{\tau \in D}\log P(\tau|\theta)=\sum_{\tau \in D}log \frac{exp(θ^{\top} f(τ))}{Z(θ)} \] 首先拆开log，得到 \[ =\sum_{\tau \in D}[θ^{\top} f(τ)-logZ(\theta)] \] 然后提取常数项，log⁡Z(θ) 与具体哪条轨迹 τ 无关，对 N 条轨迹求和就出现 N 次： \[ =θ^{\top}\sum_{\tau \in D}f(\tau)-N*logZ(\theta) \] 除以N（不影响我们求最优解），定义 \[ \tilde{f}=\frac{1}{N}\sum_{\tau \in D}f(\tau) \] 我门的目标变成如下 \[ Traget(\theta) = θ^{\top} \tilde{f} - log Z(\theta) \] 最后我们对参数θ开始求导。

第一项 \(θ^{\top} \tilde{f}\) 对 θ 求导，得到 \(\tilde{f}\)

第二项 \(\log Z(\theta)\) 用链式法则： \[ \nabla_\theta\log Z(\theta) = \frac{1}{Z(\theta)}\nabla_\theta Z(\theta) \] 我们记得 \[ Z(θ) = \sum_τ exp(θ^{\top} f(τ)) \] 因此 \[ \nabla_\theta Z(θ) = \sum_τ exp(θ^{\top} f(τ))*f(τ) \] 最终我们得到 \[ \nabla_\theta\log Z(\theta) = \sum_τ\frac{ exp(θ^{\top} f(τ))}{Z(\theta)}*f(τ) \] 我们知道前面P的定义如下 \[ P(\tau|\theta) = \exp(\theta^{\top}f(\tau))/Z(\theta) \] 因此 \[ \nabla_\theta\log Z(\theta) = \sum_τP(\tau|\theta) *f(τ)=\mathbb{E}_{P(\tau|\theta)}[f(\tau)] \] 最终两项合并，得到 \[ \nabla_\theta Target(\theta)=\tilde{f}-\mathbb{E}_{P(\tau|\theta)}[f(\tau)] \]

其中

  • \(\tilde{f}\) 是专家的轨迹链的特征均值，来自数据，因此固定不变。
  • \(\mathbb{E}_{P(\tau|\theta)}[f(\tau)]\) 表示：在当前奖励参数 \(\theta\) 下，由策略分布 \(P(\tau|\theta)\) 产生的轨迹，其特征的期望。即固定奖励网络的参数θ，用最优策略（走奖励最大的一条路）走过的所有轨迹，对他们的特征值 f(τ) 求期望。

固定当前奖励函数后，智能体会倾向于选择 reward 更高的轨迹，因此 \(\mathbb{E}_{P(\tau|\theta)}[f(\tau)]\) 可以理解为：

“当前奖励函数诱导出的策略，会产生什么样的轨迹特征”。

理论上是这样，但实际上算不出来，"解出最优策略跑无数条轨迹"在连续空间里根本算不了——轨迹是无限的。

所以我们实现的代码中用了一个近似：

​ 用当前智能体实际跑出来的轨迹，来近似"当前奖励下最优策略的轨迹"。

我们用当前 agent 输出的动作概率，跑出m条轨迹链，然后放到奖励函数中求均值，就是我们的 \(\mathbb{E}_{P(\tau|\theta)}[f(\tau)]\) 近似。

在经典 MaxEnt IRL 中： \[ R(τ)=w^⊤f(τ) \] 其中：

  • \(f(\tau)\) 是人为设计的 feature vector；
  • \(w\) 是 feature 的线性权重。

例如： \[ f(\tau)= \begin{bmatrix} \text{靠近目标}\\ \text{撞墙次数}\\ \text{步数} \end{bmatrix} \] reward 本质上是：对 feature vector 的线性投影（projection） 。即： \(\mathbb R^d \rightarrow \mathbb R\) 。

因此 \(f(\tau)\) 是多维向量，而 \(R(\tau) = w^\top f(\tau)\) 则是 feature 在方向 \(w\) 上的投影值，也就是最终 reward。

因此，我们也可以将原始的 feature matching： \[ \tilde{f}-\mathbb{E}_{P(\tau|\theta)}[f(\tau)] \] 理解为：“比较 expert 与 agent 在 feature space 中的差异”。

如果对两边同时左乘 \(w^\top\) ，则有： \[ w^\top \left( \tilde f - \mathbb E_\pi[f(\tau)] \right) \] 根据线性性质： \[ = w^\top \tilde f - w^\top \mathbb E_\pi[f(\tau)] \] 进一步得到： \[ = \mathbb E_{expert}[R(\tau)] - \mathbb E_\pi[R(\tau)] \] 现在我们的 θ 指的是 reward network 的参数。

此时：

  • 特征提取
  • 特征组合
  • reward 映射

都已经被神经网络隐式学习，因此不再显式构造 \(f(\tau)\)

更准确地说，现在实际上变成了： \[ R_\theta(s) = w^\top \phi_\theta(s) \] 其中：

  • \(\phi_\theta(s)\) 是网络内部学习到的 hidden feature；
  • \(w\) 是最后一层线性映射。

因此定义专家状态上的平均 reward： \[ R_e = R(\tau).mean() \tag{τ是所有的专家轨迹链} \] 因为奖励r 仅与状态s有关， reward_net 是逐状态打分的，因此我们也能够写成 \[ R_e = \sum_tR(s_t).mean() \]

然后固定当前 agent 参数，用 agent 采样得到 m 条轨迹，收集其中状态。对应的平均 reward 为： \[ R_a = R(\tau).mean()=\sum_tR(s_t).mean() \tag{τ是agent收集的轨迹链} \] 于是我们最终得到 reward network 的优化目标： \[ ∇Target≈R_e−R_a=R(e_s).mean()−R(a_s).mean() \] 由于我们的目标是最大化该目标，而 PyTorch 默认执行最小化，因此定义 loss： \[ loss=−(R(e_s).mean()−R(a_s).mean()) \]

这个推导里面最有意思的一点是概率与奖励的关系，即 \[ P(\tau|\theta) = \exp(\theta^{\top}f(\tau))/Z(\theta) \] 正是有该转换的基础，我们才能将奖励与概率之间的相互影响写出来。因此如果我们想探索 结果A与影响因子B之间的关系，我们可以去找找现有的公式或者自行构造一个合理的公式。

3. 对抗学习(Generative Adversarial Imitation Learning)

3.1 理论介绍

GAIL（Generative Adversarial Imitation Learning）= 对抗式模仿学习

核心思想借鉴自 Generative Adversarial Network：

有两个网络在“对抗”：

  • 策略 π（Generator）：产生行为（像专家一样行动）
  • 判别器 D（Discriminator）：判断行为是“专家的”还是“模型的”

判别器任务是学会区分专家和智能体，可以输入专家的(s,a)与agent的(s,a)做交叉熵

在收集数据过程中，我们将判别器的结果修改成奖励 \[ r_{GAIL}=\log D(s,a)−\log (1−D(s,a)) \]

当前动作越接近专家，则奖励越高。即，gail_r 把"像专家的行为"映射成高奖励。

Policy 通过 PPO 更新，趋向高奖励路径。最终判别器无法区分两者，是收敛的标志。

收敛时 D(s,a)≈0.5 。此时 \[ r_{GAIL}=\log0.5−\log0.5=0 \] 收敛意味着智能体行为已经和专家一样，判别器"放弃区分"，此时奖励信号自然归零——任务完成了，不再需要额外激励。

4. 对抗逆强化学习（Adversarial Inverse Reinforcement Learning）

4.1 理论

在之前的girl中，我们的判别器仅仅用来判断是否是专家数据。比如输入专家数据 (s,a) ，判别器就将输出往1方向拟合。如果是agent数据 (s,a)，则往0方向拟合。

GAIL 的判别器就是一个纯粹的二分类器： \[ D(s,a)=σ(f(s,a)) \] f(s,a) 是任意神经网络，没有任何结构约束，只负责分类，学到的东西无法解释。

在 AIRL 中，AIRL 给判别器加了结构约束，强制它的形式为： \[ D_\theta(s,a,s')=\frac{\exp(f_\theta(s,a,s'))}{\exp(f_\theta(s,a,s'))+\pi(a|s)} \] 其中\(f_\theta\) 被进一步分解为：

我们将 f 强制约束成 优势计算的结构。

我们判别器D的训练目标和 GAIL 一样，做二分类：

对于专家数据的(s,a,s')，将其拟合成1，而对于agent轨迹的(s,a,s')，将其拟合为0.

当收敛时，策略 π 完全模仿专家时，判别器无法区分两者，此时： \[ D_\theta(s,a,s')=\frac{\exp(f_\theta(s,a,s'))}{\exp(f_\theta(s,a,s'))+\pi(a|s)}=0.5 \] 即判断为agent和专家的概率都为50%，无法区分开来。

此时 \[ \exp(f)=π(a|s) \] 即 \[ f(s,a,s')=\logπ(a|s) \] 为什么这意味着 \(f→A^∗\)

最优策略满足： \[ π^∗(a∣s)∝exp(A^∗(s,a)) \] 即，在最优策略中，策略在某状态s下的所有动作a概率，与 对应优势的exp值成正比 \(exp(A^∗(s,a))\)

实际上：

举例：

接下来对 \(\pi^*(a|s)\) 两边取 log⁡：

即： \[ \logπ^∗(a∣s)=A^∗(s,a)+C(s) \] 其中 C(s) 是只和状态有关的归一化常数，会被 \(h_\phi(s)\) 吸收。

所以收敛时： \[ f^∗(s,a,s')=\logπ^∗(a|s)=A^∗(s,a) \] 因为我们强制 \(f=g_θ(s,a)+γh_ϕ(s')−h_ϕ(s)\)，

因此一组可行解是： \[ g_θ=r^∗,h_ϕ=V^∗ \] 但该解不是唯一的，因为 \(h_\phi(s)\) 只刻画“状态之间的相对差异”，而不是绝对数值。只要同时对 reward 和 h 做一致的调整（相当于给状态加一个势能函数），最终结果不会发生变化。

总之，最终我们学到了我们需要的奖励函数 \(g_\theta\) ，它真正解释了我们做哪些动作有什么奖励。且与环境无关。（奖励函数的值并一定和真实奖励相同，我们需要的是“能解释行为且不依赖环境的最简 reward 表达）

5. AIRL的迁移学习

AIRL 学出来的奖励函数，相比于 IRL 更具解释性，因为它排除了环境动力学的影响。

IRL 在训练时隐式依赖了当前环境的转移动力学 \(p(s'∣s,a)\) ，奖励与特定环境耦合。比如同样是拿钥匙，在迷宫A里学到的奖励，换到迷宫B（地形不同，动力学不同）就失效了——IRL 学到的其实是"在这个环境的这个位置做这个动作有奖励"。

AIRL 通过构造 \(f_\theta(s,a)\) 把动力学项约掉，学到的是"拿钥匙这个行为本身有奖励"，与在哪个环境、怎么转移无关，因此可以迁移到不同环境，这才是解释性更强的真正含义。

所以，我们这里将在上一步5x5环境中学到的agent迁移到8x8的环境中。

但毕竟是5x5的小环境到8x8的大环境，在小环境中基本上钥匙跟门都是在视野内的，大环境中则可能出现丢失的情况。因此agent也需要同步进行训练。

我们的方法是，agent在8x8环境中按照学好的策略收集数据。将成功的数据作为专家经验，重新训练判别器。同时，我们只要前30个步数最短的成功路径来当作我们的专家经验（希望学到最优策略）。
